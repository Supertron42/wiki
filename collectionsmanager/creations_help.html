Creations Tab - Creating & Sharing Collections

The Creations tab allows you to export your modlist as a shareable JSON file, import previously exported modlists, and manage FOMOD installer choices. This is useful for:

  • Sharing your curated mod setup with friends
  • Creating backups of your collection configuration
  • Building offline-friendly modlists
  • Preserving FOMOD choices for reinstallation
Note: The Creations tab visibility can be toggled in Options. See Toggling Creations Tab for details.

Toggling the Creations Tab

The Creations tab visibility can be controlled in the Options:

  1. Go to the Options tab
  2. Click Show next to "Developer Options"
  3. Toggle the Show Creations Tab checkbox
Tip: The Creations tab setting is saved automatically and will persist between sessions.

Exporting a Modlist

Export your current collection as a JSON file that can be shared or imported later.

How to Export

  1. Load a collection or have mods in your mod list
  2. Go to the Creations tab
  3. Review the mod table to see what will be exported
  4. Optionally configure mod flags (essential, optional, external, bundle)
  5. Click Export Modlist
  6. Choose a save location and filename

What's Included in the Export

  • Mod metadata: Name, version, file ID, mod ID
  • Install order: The sequence mods should be installed
  • FOMOD choices: Saved installer selections
  • Mod flags: Essential, optional, external, or bundle status
  • Collection info: Author name, version, description
Note: The export does NOT include the actual mod files, only the metadata needed to download and install them.

Exporting FOMOD Choices

You can export just your FOMOD installer choices separately from the full modlist. This is useful for:

  • Backing up your installer preferences
  • Sharing specific FOMOD configurations
  • Transferring choices to another profile

How to Export FOMOD Choices

  1. Go to the Creations tab
  2. Click Export FOMOD Choices
  3. Choose a save location

The exported file contains all your saved FOMOD selections in JSON format.

Export File Format

Exported modlists use a JSON format compatible with Collections Manager. The structure includes:

FieldDescription
nameCollection name
authorAuthor name
versionCollection version string
descriptionCollection description
gameTarget game (e.g., "skyrimspecialedition")
modsArray of mod objects with metadata
fomod_choicesFOMOD installer selections per mod

Mod Object Fields

FieldDescription
nameMod display name
mod_idNexus mod ID
file_idNexus file ID
versionFile version
optionalWhether the mod is optional
externalWhether downloaded from external source
bundleWhether included in a bundle archive

Importing a Modlist

Import a previously exported modlist JSON file to recreate a collection.

How to Import

  1. Go to the Creations tab
  2. Click Import Modlist
  3. Select the JSON file to import
  4. The collection will be loaded into Collections Manager
  5. You can then download and install the mods normally
Important: Importing creates a new collection entry. Make sure you're importing into the correct MO2 profile.

Import Process Details

When you import a modlist, Collections Manager:

  1. Parses the JSON file and validates the format
  2. Generates a collection.json compatible with the download engine
  3. Stores FOMOD choices for automatic application during install
  4. Adds the collection to your Collections tab history
  5. Switches to the Mods tab ready for installation

After Import

  • Click Begin Install (Premium) or Open Download Links (Free) to start
  • Mods will download and install in the specified order
  • FOMOD choices will be automatically applied

Mod Table Overview

The Creations tab displays a table of all mods in your current collection with their export settings.

Table Columns

Sorting and Filtering

  • Click column headers to sort
  • Use the search box to filter mods by name
  • Right-click for context menu options

Mod Flags

Each mod can have flags that affect how it's handled during installation:

FlagIconDescription
EssentialStarRequired mod - cannot be skipped
OptionalQuestion markUser will be prompted to install or skip
ExternalGlobeDownloaded from external source (not Nexus API)
BundlePackageIncluded in a bundle archive with the collection

Setting Flags

  1. Select one or more mods in the table
  2. Right-click to open the context menu
  3. Choose the flag to toggle
Tip: Flag changes are saved automatically and included in exports.

Configuring FOMOD Choices

You can pre-configure FOMOD installer choices before exporting:

How to Configure

  1. Select a mod with a FOMOD installer
  2. Click Configure FOMOD or right-click and select it
  3. A dialog appears showing available options
  4. Select your preferred choices
  5. Click Save

These choices will be:

  • Automatically applied when installing the mod
  • Included in modlist exports
  • Transferred when sharing the collection

Sharing Collections with Others

To share your collection with friends or the community:

Step-by-Step

  1. Set up your modlist exactly how you want it
  2. Configure FOMOD choices for complex installers
  3. Mark optional mods appropriately
  4. Export the modlist to a JSON file
  5. Share the JSON file (Discord, email, cloud storage, etc.)

What Recipients Need

  • MO2 with Collections Manager installed
  • Nexus Mods account (Premium recommended for auto-download)
  • The same game installed
Best Practice: Include a README with your export explaining any manual steps, recommended settings, or mod conflicts to watch for.

Backup and Restore

Use the Creations tab to backup your collection configuration:

Creating a Backup

  1. Load your collection
  2. Export the modlist with all FOMOD choices
  3. Save to a backup location (cloud storage recommended)

Restoring from Backup

  1. Create a fresh MO2 profile
  2. Import the modlist JSON
  3. Run the installation process
Note: Backups don't include actual mod files. You'll need to re-download everything. For full offline backups, consider creating a bundle.

Creating Offline Collections

For collections that don't require internet to install, you can create bundles:

Bundle Features

  • Include mod archives directly in the collection
  • Add INI tweaks that apply automatically
  • Include binary patches (using bsdiff4)
  • Everything needed in a single archive

Creating a Bundle

  1. Export your modlist
  2. Mark mods as "Bundle" that should be included
  3. Gather the mod archives
  4. Package with the collection.json
  5. Compress as .7z archive
Advanced: Bundle creation is a manual process. See the Tools tab for the Bundle Installation tool that handles extraction and patching.

Tips for Sharing

  • Test your exported collection by importing it into a fresh profile
  • Document any mods that require special handling
  • Include version numbers so users know what they're getting
  • Consider creating separate "core" and "optional" exports for large collections
