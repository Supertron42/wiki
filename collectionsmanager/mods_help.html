Purpose of the Mods Tab

The Mods tab is the main interface for viewing and installing mods from a Nexus collection. It shows all mods in the collection organized by section (Essential, Optional, External) and tracks the status of each mod through the download and installation process.

Key Features:
  • View all mods in the loaded collection
  • Download and install mods automatically (Premium) or semi-manually (Free)
  • Track progress with real-time status updates
  • Skip individual mods you don't want
  • Handle external mods from non-Nexus sources

Loading a Collection

Before using the Mods tab, you need to load a collection:

  1. Paste a Nexus collection URL in the URL field at the top
  2. Click Load
  3. Select a revision from the dropdown (if multiple exist)
  4. The Mods tab will populate with all mods from the collection

URL Formats Supported

  • https://next.nexusmods.com/skyrimspecialedition/collections/abc123
  • https://www.nexusmods.com/skyrimspecialedition/collections/abc123
  • nexusmods.com/skyrimspecialedition/collections/abc123

Mod Sections

Collections organize mods into sections that are processed in order:

Essential Mods

Required mods that the collection author considers necessary. These are installed first and cannot be skipped by default.

Optional Mods

Recommended but not required mods. After essentials complete, you'll be prompted to continue with optionals. You can skip individual optional mods.

External Mods

Mods from sources other than Nexus (Patreon, LoversLab, direct downloads, etc.). These require manual download - the plugin will open the download page for you.

Section headers appear as colored rows in the table to help you identify where each section begins.

Premium Mode Installation

If you have Nexus Premium, downloads happen automatically via the API.

How to Install

  1. Load a collection
  2. Click Begin/Resume Install
  3. The plugin downloads and installs mods one at a time
  4. FOMOD installers are handled automatically (if choices are saved)
  5. Progress is shown in the status bar

Download Ahead

The plugin can pre-download upcoming mods while the current one installs. Configure this in the Options tab (Download Ahead setting).

Tip: For large collections, let it run unattended. The plugin handles everything automatically and resumes where it left off if interrupted.

Free Mode Installation

Without Nexus Premium, you need to click download buttons on the Nexus website.

How to Install

  1. Load a collection
  2. Click Open Download Links to open mod pages in your browser
  3. Click the download button on each Nexus page
  4. MO2 will catch the downloads automatically
  5. Once mods are downloaded, click Install Downloaded

Max Tabs Setting

Control how many browser tabs open at once with the "Max tabs" spinner. Opening too many can overwhelm your browser.

Note: Free mode is slower but still automates the installation process once archives are downloaded.

Skipping Mods

Use the Skip checkbox to exclude mods from installation:

  • Skipped mods won't be downloaded or installed
  • Skip status is saved and persists between sessions
  • You can unskip mods later and install them

When to Skip

  • You already have a mod installed from another source
  • You don't want a particular optional mod
  • A mod conflicts with something else you're using
  • You want to install a different version manually

Right-Click Context Menu

Right-click selected mods for additional actions:

Download Actions

  • Download Selected - Download only selected mods (Premium)
  • Open Download Links - Open Nexus pages for selected mods (Free)
  • Retry Failed Downloads - Retry mods that failed to download
  • Rescan / Fix Stuck - Check if a stuck download actually completed

Install Actions

  • Install Selected - Install selected downloaded mods

Navigation

  • Open Mod Pages - Open the Nexus mod page (not download page)
  • Copy Links - Copy download URLs to clipboard

Skip Management

  • Toggle Skip - Flip skip status for selected mods
  • Remove 'Skip' From All - Unskip all mods in the collection

Row Actions

Each row has action buttons in the rightmost column:

Open Button

Opens the mod's Nexus page in your browser. Useful for reading descriptions, checking requirements, or reporting issues.

Install Button

Appears when the mod's archive is downloaded. Click to install that specific mod immediately.

Search & Filter

Search Box

Type to filter mods by name. The filter applies instantly as you type. Clear the search to show all mods again.

Status Filter Dropdown

  • All - Show all mods
  • Pending - Mods waiting to be processed
  • Downloading - Currently downloading
  • Downloaded - Ready to install
  • Installed - Successfully installed
  • Failed/Stuck - Problem mods needing attention
  • Skipped - Mods you've chosen to skip
Use filters to focus on specific mods, like finding all failed downloads or checking what's left to install.

External Mods

External mods come from sources other than Nexus Mods:

  • Patreon-exclusive mods
  • LoversLab mods
  • Direct download links
  • Other mod hosting sites

How to Handle External Mods

  1. Click Open External Links to open their download pages
  2. Download the files manually from each site
  3. Place the archives in MO2's downloads folder
  4. The plugin will detect them and mark them as downloaded
  5. Continue with installation as normal
Note: External mods may require accounts on other sites or may be behind paywalls. The collection author should provide instructions.

Bundle Files

Some collections include bundle archives containing:

  • Patches for mod compatibility
  • Configuration files (INI tweaks, MCM settings)
  • Custom assets the author created
  • Tools or utilities

Bundle Panel Interface

After all regular mods are processed, a bundle panel appears on the left side showing a categorized preview of bundle contents:

  • Plugins - Plugin files (.esp, .esm, .esl)
  • Archives - BSA/BA2 archives
  • INI Files - Configuration files for your mod folder
  • Patches - Compatibility patches (shown with installation status)
  • INI Tweaks - Changes to game INI files
  • Other Files - Any other bundled content

INI Tweaks Preview

If the bundle includes INI tweaks, click Preview INI Changes... to see exactly what will be modified in your game INI files. The preview shows:

  • Target INI file (e.g., Skyrim.ini, SkyrimPrefs.ini)
  • Section and setting names
  • Current values vs new values
  • Changed values are highlighted

Installing Bundles

  1. Review the content preview in the panel
  2. Click Install Bundle to install all applicable content
  3. Or click Skip Bundle to complete without bundle files
Patches for mods you haven't installed are automatically skipped. This prevents orphaned patches that would cause errors.

Troubleshooting

Download Stuck at 0 KB

  1. Go to MO2's Downloads tab (right panel)
  2. Find the stuck download
  3. Right-click > Pause, then Resume
  4. Return to Collections Manager and click Begin/Resume

Download Won't Start

  • Check your API key in the Options tab
  • Verify you're logged into Nexus in your browser
  • Check if Nexus is having server issues
  • Try the download manually from the mod page

Mod Shows Wrong Status

  • Use Tools > Maintenance > Rebuild Index
  • Right-click the mod > Rescan / Fix Stuck
  • Check if the archive exists in MO2's downloads folder

FOMOD Choices Not Applied

  • The mod may not have saved choices in the collection
  • Check the FOMODs tab to see saved choices
  • Install manually once to capture your preferred choices

Best Practices

  • Let it run - Don't interrupt the installation process unnecessarily
  • Check logs - MO2's log panel shows what's happening
  • Handle failures gracefully - Use Retry Failed for problem downloads
  • Skip wisely - Only skip mods you're sure you don't need
  • Install bundles last - Wait until main mods are done
  • Run maintenance - If status seems wrong, rebuild the index
  • Save your progress - The plugin remembers what's installed, so you can resume anytime

For Large Collections (500+ mods)

  • Close unnecessary programs to free memory
  • Expect the process to take several hours
  • You can close and reopen MO2 - progress is saved
  • Consider doing it in stages using Skip to break into batches
