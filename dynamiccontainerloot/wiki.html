Dynamic Container Loot Wiki

An SKSE plugin that consolidates container loot from all mods at runtime. When multiple mods edit the same container, Skyrim's last-loaded-wins rule discards changes from all but the final override. Dynamic Container Loot merges the lost items back in automatically - no patches, no manual conflict resolution.

Quick Start: Install via your mod manager and launch the game. The plugin runs automatically on data load. Check Data/SKSE/Plugins/DynamicContainerLoot.log to see what was merged.

The Problem

Skyrim uses a "last writer wins" rule for records. If three mods edit the same chest (e.g. BarrelFood01), only the last mod in your load order keeps its changes. Items added by the other two mods are silently lost.

The Solution

Dynamic Container Loot reads the original container definition from every plugin that touches it, computes what each "loser" mod added, removed, or changed, and merges those deltas into the winning record at runtime. The result is the same as if you had built a manual Bash Patch - but fully automatic.

How It Works

  • Fires on kDataLoaded - runs once, before the main menu
  • Scans every CONT (container) form for multi-source conflicts (3+ plugins touching the same record)
  • Parses each plugin's CONT subrecords directly from disk to reconstruct what each mod intended
  • Computes per-mod deltas against the base record: additions, removals, and count changes
  • Applies merged deltas to the winning record in memory

Key Features

Fully automatic
No ESP required
Works with all mods
SE + AE + VR support
Additions merged
Removals respected
Count changes applied
Conflict-aware (winner preserved)

INI Settings

Configuration is at Data/SKSE/Plugins/DynamicContainerLoot.ini. If the file doesn't exist, it will be auto-generated with defaults on first launch.

[General]
;Enable detailed merge logging (true/false)
enablelogs = false

Settings Reference

enablelogs

Default: false
When enabled, logs every individual item added, removed, or changed per container. Useful for debugging but adds startup time due to EditorID lookup.

Reading the Log

The log at Data/SKSE/Plugins/DynamicContainerLoot.log always shows a summary:

=== DynamicContainerLoot: Merge complete ===
  Total containers: 1247
  With overrides (base + winner only): 89
  With conflicts (3+ sources): 34
  Actually merged: 12
  Total items added: 27
  Total items removed: 3
  Total items count changed: 2

With enablelogs=true, each merged container also gets detailed output:

  BarrelFood01 [CONT:0001003E]
    Sources: Skyrim.esm, FoodExpansion.esp, LootOverhaul.esp
    -- Changes --
      + [ALCH] Apple x2 [00064B2F] (added by FoodExpansion.esp)
      + [ALCH] SweetRoll x1 [00064B40] (added by FoodExpansion.esp)
    -- Carried forward --
      . [ALCH] Bread x1 [00064B30]
    Result: +2 added, -0 removed, ~0 count changed
    -- Final state --
      * [ALCH] Bread x1 [00064B30]
      * [ALCH] Apple x2 [00064B2F]
      * [ALCH] SweetRoll x1 [00064B40]
    (3 items total)

Performance note: Detailed logging requires scanning every loaded mod's LVLI records from disk to build an EditorID cache (so leveled list items get readable names in the log). This adds noticeable startup time. Keep enablelogs=false for normal play.

Compatibility

  • Works alongside Wrye Bash / Mator Smash patches - they handle the same problem at build time, this handles it at runtime for anything they miss
  • Safe with any load order - reads plugin data non-destructively
  • No conflicts with other SKSE plugins - doesn't hook any engine functions

Requirements

  • Skyrim SE (1.5.97+), AE (1.6.x), or VR
  • SKSE64 (matching your game version)
  • Address Library for SKSE Plugins
