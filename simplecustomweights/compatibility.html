Compatibility

Simple Custom Weights uses runtime patching, which makes it highly compatible with other mods.

How It Works

  • Modifies in-memory item weights after game data loads
  • Does not edit plugins on disk — no ESP/ESM changes
  • No xEdit patches required
  • Works with vanilla and modded items automatically

Key benefit: Because this runs at runtime, it works with any mod load order without conflicts or patches.

Load Order & Conflicts

  • Runtime patching can override weights set by ESP, ESL, or ESM plugins
  • If another SKSE plugin also changes weights at runtime, whichever runs last wins for that form
  • On save load, Simple Custom Weights reapplies its values, so it typically takes precedence

Items That Are Skipped

The following items are never modified to prevent breaking quests or game mechanics:

  • Quest items — Items with the Quest Item flag set
  • VendorNoSale items — Items with the VendorNoSale keyword

Note: Gold is handled explicitly with its own setting and is not affected by the blacklist logic.

Modded Content

The plugin identifies item categories using vanilla vendor keywords:

  • VendorItemFood — Cooked food and drinks
  • VendorItemFoodRaw — Raw food
  • VendorItemTool — Tools
  • Book and note keywords for reading materials

Tip: Modded items that use these vanilla keywords will be categorized correctly. Items with custom keywords may fall into the generic misc handling instead.

Known Compatible Mods

  • Complete Alchemy and Cooking Overhaul (CACO) — Works with CACO's items
  • Legacy of the Dragonborn — Museum items are typically quest items and will be skipped
  • Ordinator / Vokrii — Perks that affect carry weight work normally
  • SkyUI / MCM — Can be used to trigger reloads via Papyrus

Potential Conflicts

  • Other SKSE plugins that modify item weights at runtime may have unpredictable behavior depending on load order
  • Mods that expect specific item weights for gameplay mechanics may behave differently
