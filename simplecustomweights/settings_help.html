INI Settings

All settings are read from Data/SKSE/Plugins/SimpleCustomWeights.ini under the [Settings] section.

Weight Multipliers

These settings multiply the base weight of items. A value of 0.5 means half weight, 1.0 means unchanged.

Setting Default Description
WeaponWeight 1.0 Multiplier for all weapons
ArmorWeight 1.0 Multiplier for all armor pieces
PotionPoisonWeight 0.5 Multiplier for potions and poisons
ScrollWeight 0.1 Multiplier for scrolls
FoodDrinkWeight 0.5 Multiplier for cooked food and drinks (VendorItemFood)
RawFoodWeight 0.5 Multiplier for raw food (VendorItemFoodRaw)
BookWeight 0.5 Multiplier for books, notes, and recipes
IngredientWeight 0.1 Multiplier for alchemy ingredients
SoulGemWeight 0.1 Multiplier for soul gems
MiscWeight 0.1 Multiplier for miscellaneous items (excluding gold)
ToolWeight 0.1 Multiplier for tool items (VendorItemTool keyword)

Fixed Weight Overrides

These settings set a specific weight value instead of multiplying. Use -1 to disable (leave unchanged).

Setting Default Description
GoldWeight 0.0 Fixed weight for gold (always applied)
KeyWeight -1.0 Fixed weight for keys. Set to 0+ to enable, -1 to disable.
LockpickWeight -1.0 Fixed weight for lockpicks. Set to 0+ to enable, -1 to disable.

Safety Settings

Setting Default Description
ApplyMinimumWeightToZero true Prevents items from becoming truly weightless
MinimumNonZeroWeight 0.1 Minimum weight when clamping is enabled

Warning: Setting MinimumNonZeroWeight lower than 0.01 will generate a log warning. Extremely tiny weights can cause odd behavior in some game calculations.

Example INI

[Settings]
; Weight Multipliers
WeaponWeight=1.0
ArmorWeight=1.0
PotionPoisonWeight=0.5
ScrollWeight=0.1
FoodDrinkWeight=0.5
RawFoodWeight=0.5
BookWeight=0.5
IngredientWeight=0.1
SoulGemWeight=0.1
MiscWeight=0.1
ToolWeight=0.1

; Fixed Overrides
GoldWeight=0.0
KeyWeight=-1.0
LockpickWeight=-1.0

; Safety
ApplyMinimumWeightToZero=true
MinimumNonZeroWeight=0.1

Notes

  • KeyWeight and LockpickWeight use -1 as a "disabled" value. Any value 0 or higher enables fixed weight behavior.
  • GoldWeight always applies a fixed weight to gold â€” it does not use a multiplier.
  • Food, raw food, and tools are identified by vanilla vendor keywords. Modded items without these keywords may fall into the misc category.
  • All settings are logged on startup so you can verify what values were loaded.
