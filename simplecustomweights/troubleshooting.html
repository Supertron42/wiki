Troubleshooting

Common issues and how to resolve them.

Nothing changes in game

Check the DLL location:

  • Verify the DLL is at Data/SKSE/Plugins/SimpleCustomWeights.dll

Check the log file:

  • Open SimpleCustomWeights.log in your SKSE log directory
  • Look for DataLoaded and PostLoadGame messages
  • If these are missing, SKSE may not be loading the plugin

Check the INI path:

  • Confirm the INI is at Data/SKSE/Plugins/SimpleCustomWeights.ini
  • If the INI fails to load, the log will show a failure message

Some items change but others don't

These items are intentionally skipped:

  • Quest items (items flagged as Quest Item)
  • Items with the VendorNoSale keyword

These require specific settings:

  • Keys only change if KeyWeight is set to 0 or higher
  • Lockpicks only change if LockpickWeight is set to 0 or higher

Category detection:

  • Food, raw food, tools, and some book types depend on vanilla vendor keywords
  • Items without those keywords may be handled as misc instead

Items become weightless and something feels off

  • Make sure ApplyMinimumWeightToZero=true is set in your INI
  • This prevents items from reaching true zero weight
  • If you want near-zero weights, use a small MinimumNonZeroWeight like 0.01
  • Don't go smaller than 0.01 â€” extremely tiny weights can cause odd behavior

How do I verify what settings were loaded?

  • Open SimpleCustomWeights.log
  • The plugin logs every loaded setting value on startup
  • You'll see lines like WeaponWeight = 1.0 for each setting
  • Patch summaries show how many items were modified per category

Changes don't apply after editing the INI

  • Settings reload automatically when you load a save
  • For immediate effect without loading a save, use the Papyrus reload function
  • Make sure you saved the INI file before loading the save

Getting Help

When reporting issues, include your SimpleCustomWeights.log file and your SimpleCustomWeights.ini contents. This helps diagnose problems quickly.
